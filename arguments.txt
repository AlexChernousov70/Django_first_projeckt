Одинарный индекс Ё необходим для сортировки отзывов, которые достаются из базы данных каждый раз при загрузке главной страницы, подразумевается что количество отзывов у нас будет большое. Мастеров будет мало поэтому их не индексируем.

Одинарный индекс сортировки заказов необходим, так как каждый раз при открытии списка заявок у нас будет происходить их сортировка оп убыванию. Подразумевается, что в дальнейшем заказов у нас будет большое количество.

Составной индекс поиска по телефону + сортировка по убыванию заказов необходим потому что поиск по телефону - основной поиск заяки. 
Администратору звонят клиенты "Здравствуйте, у нас такой-то заказ" и будет поиск по телефону. А сортировка по убыванию у нас перманентна.
